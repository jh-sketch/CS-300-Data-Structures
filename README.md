# CS-300-Data-Structures
## Reflection

### What was the problem you were solving in the projects for this course?
The problem I was solving in this course was finding logical and efficient ways to store and manage data in a program. I had to figure out how to organize course information so that it could be searched, sorted, and displayed correctly.

### How did you approach the problem? Consider why data structures are important to understand.
The way I approached the problem was by using algorithms for inserting data, removing data, finding data, and sorting data within each data structure. I compared how vectors, hash tables, and binary search trees handled these operations.

Data structures are important to understand because they directly affect how a program performs. I was able to see how different structures impact speed and memory usage. They also determine how easy it is to manage and access data, which is important when working with larger programs.

### How did you overcome any roadblocks you encountered while going through the activities or project?
One of the biggest roadblocks I ran into was getting the binary search tree logic working correctly. It took some time to fully understand how nodes are inserted and how the traversal works, especially making sure the courses printed in the correct order.

I also had issues with compiling errors and debugging small mistakes, like incorrect comparisons or pointer issues. To overcome this, I went step by step, tested my code often, and used trial and error to figure out what was going wrong. Breaking the problem into smaller parts made it easier to fix issues and understand what the code was doing.

### How has your work on this project expanded your approach to designing software and developing programs?
This project changed the way I think about designing programs. Before, I focused more on just getting the code to work. Now I think more about how the data is structured first, and how that will affect performance.

I also started thinking more about efficiency and scalability. Instead of just writing code, I now consider which data structure makes the most sense based on the problem.

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
This project helped me improve how I organize my code. I started breaking my program into smaller functions and keeping things more structured, which makes it easier to read and maintain.

I also became more careful with naming variables and writing code in a way that is easier to understand later. This makes it easier to go back and fix bugs or add new features without having to rewrite everything. Overall, I feel more confident writing code that is cleaner and easier to work with.
